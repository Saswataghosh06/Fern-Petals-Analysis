# Data Dictionary — FNP Sales Analysis

## Overview
This document describes all tables and columns across the three layers
of the FNP data warehouse (Bronze, Silver, Gold).

---

## Source Files (Raw CSV)

| File | Rows | Description |
|------|------|-------------|
| orders.csv | 1,000 | Transactional order data |
| products.csv | 70 | Product catalog |
| customers.csv | 100 | Customer master data |

---

## Silver Layer — Cleaned and Typed

### silver.orders
| Column | Type | Source | Description |
|--------|------|--------|-------------|
| order_id | NVARCHAR(50) | bronze.orders | Unique order identifier |
| customer_id | NVARCHAR(50) | bronze.orders | References customer |
| product_id | NVARCHAR(50) | bronze.orders | References product |
| quantity | INT | bronze.orders | Units ordered — cast to INT |
| order_date | DATE | bronze.orders | Cast to DATE type |
| order_time | TIME | bronze.orders | Cast to TIME type |
| delivery_date | DATE | bronze.orders | Cast to DATE type |
| delivery_time | TIME | bronze.orders | Cast to TIME type |
| delivery_days | INT | **Engineered** | DATEDIFF(day, order_date, delivery_date) |
| order_hour | INT | **Engineered** | DATEPART(hour, order_time) |
| order_day | NVARCHAR(20) | **Engineered** | DATENAME(weekday, order_date) |
| order_month | NVARCHAR(20) | **Engineered** | DATENAME(month, order_date) |
| location | NVARCHAR(100) | bronze.orders | Trimmed whitespace |
| occasion | NVARCHAR(100) | bronze.orders | UPPER + TRIM standardized |
| revenue | DECIMAL(10,2) | **Engineered** | quantity × price_inr |

### silver.products
| Column | Type | Source | Description |
|--------|------|--------|-------------|
| product_id | NVARCHAR(50) | bronze.products | Unique product identifier |
| product_name | NVARCHAR(255) | bronze.products | Trimmed whitespace |
| category | NVARCHAR(100) | bronze.products | Trimmed whitespace |
| price_inr | DECIMAL(10,2) | bronze.products | Cast from NVARCHAR to DECIMAL |
| occasion | NVARCHAR(100) | bronze.products | UPPER + TRIM standardized |
| description | NVARCHAR(500) | bronze.products | Trimmed, NULLs replaced with empty string |

### silver.customers
| Column | Type | Source | Description |
|--------|------|--------|-------------|
| customer_id | NVARCHAR(50) | bronze.customers | Unique customer identifier |

| Column | Type | Description |

## Engineered Columns Summary

These columns do not exist in the source CSV files.
They were derived during the Silver layer transformation.

| Column | Table | Formula |
|--------|-------|---------|
| delivery_days | silver.orders | DATEDIFF(day, order_date, delivery_date) |
| order_hour | silver.orders | DATEPART(hour, order_time) |
| order_day | silver.orders | DATENAME(weekday, order_date) |
| order_month | silver.orders | DATENAME(month, order_date) |
| revenue | silver.orders | quantity × price_inr (joined from silver.products) |

## Data Quality Notes

| Issue | Layer Found | Resolution |
|-------|------------|------------|
| contact_number loaded as BIGINT | Bronze | Cast to NVARCHAR in Silver — preserves leading zeros |
| order_id loaded as SMALLINT | Bronze | Cast to NVARCHAR in Silver — IDs are identifiers not numbers |
| Gender values not standardized | Bronze | Standardized to Male/Female in Silver using CASE |
| UTF-8 BOM encoding on CSV files | Source | Confirmed clean UTF-8 before import |
| BULK INSERT service account restriction | Bronze | Used SSMS Import Flat File Wizard — documented in script headers |
